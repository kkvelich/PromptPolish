# PromptPolish

iOS app + keyboard extension that turns rough voice dictation into clean, well-structured AI prompts. Powered by the Claude API with prompt caching.

## What you get

- **Host app** — record-and-polish scratchpad with a copy button. Holds your Anthropic API key (in Keychain, shared with the keyboard via App Group).
- **Keyboard extension** — "✨ Polish" key sits above the system keyboard. Inside any text field, dictate or type a rough idea, tap Polish, and the text is replaced in place with a clean prompt. Includes Undo.

## Prerequisites

1. **Full Xcode** from the Mac App Store. Command Line Tools alone is not enough.
2. After Xcode installs, point `xcode-select` at it:
   ```sh
   sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
   xcodebuild -version   # should print Xcode 15+
   ```
3. **Anthropic API key** — get one at https://console.anthropic.com → Settings → API Keys. Add ~$5 in credits; it lasts months for personal use.
4. A free Apple ID is sufficient for installing on your own iPhone (no $99 dev account required).

## First-time project setup

The Xcode project is generated from `project.yml` via XcodeGen.

```sh
# Regenerate after editing project.yml or moving files around
xcodegen generate
```

Open `PromptPolish.xcodeproj` in Xcode.

### Signing (one-time per machine)

You need to change the bundle ID prefix from `com.kiranvelicheti.promptpolish` to something unique to you, otherwise Apple will reject the provisioning profile (the IDs are namespaced per developer account).

1. In Xcode, select the **PromptPolish** project (top of the file tree) → **PromptPolish** target → **Signing & Capabilities**.
2. Change **Bundle Identifier** to `com.<yourname>.promptpolish`.
3. Set **Team** to your Apple ID (sign in via Xcode → Settings → Accounts if you haven't).
4. Repeat for the **PromptPolishKeyboard** target — bundle ID should be `com.<yourname>.promptpolish.keyboard`.
5. Both targets need the **App Groups** capability with a group identifier like `group.com.<yourname>.promptpolish`. The entitlements files reference `group.com.kiranvelicheti.promptpolish` — search/replace if you renamed.
6. Also update `Shared/AppGroup.swift` and `Shared/KeychainHelper.swift` (search for `kiranvelicheti`) to match your new App Group ID.

A cleaner alternative: edit `project.yml` once with your prefix and re-run `xcodegen generate`.

### Building to your iPhone

1. Plug in your iPhone, trust the Mac when prompted.
2. In Xcode's device selector (top center), pick your iPhone.
3. Cmd+R to build and run. The host app launches on your phone.
4. First time: iOS asks you to trust the developer profile — Settings → General → VPN & Device Management → tap your Apple ID → Trust.

### Enabling the keyboard

1. On your iPhone: Settings → General → Keyboard → Keyboards → **Add New Keyboard**.
2. Pick **PromptPolish** under "Third-Party Keyboards."
3. Tap the newly added PromptPolish entry → enable **Allow Full Access**. This is required for the keyboard to make network calls to Claude. Accept the warning.

### First-run test

1. Open any app with a text field (Notes is great).
2. Tap the globe key on the keyboard until **PromptPolish** is selected.
3. Switch back to the regular keyboard, dictate a messy sentence ("um i need to like write an email to my landlord about the leak"), switch to PromptPolish, tap **✨ Polish**.
4. Watch the text be replaced with a clean structured prompt.
5. Use **Undo** if you want the original back.

## How it works

```
PromptPolish/
├── project.yml                          # XcodeGen spec — source of truth for project structure
├── PromptPolish/                        # Host app target
│   ├── PromptPolishApp.swift            # App entry + onboarding gate
│   ├── ContentView.swift                # Record / transcript / improve / output
│   ├── SettingsView.swift               # API key + model picker
│   ├── OnboardingView.swift             # First-run setup walkthrough
│   ├── SpeechService.swift              # SFSpeechRecognizer wrapper
│   ├── Info.plist                       # Generated by XcodeGen from project.yml
│   └── PromptPolish.entitlements        # App Group membership
├── PromptPolishKeyboard/                # Keyboard extension target
│   ├── KeyboardViewController.swift     # Polish bar + UITextDocumentProxy logic
│   ├── Info.plist                       # Generated; declares RequestsOpenAccess
│   └── PromptPolishKeyboard.entitlements
└── Shared/                              # Compiled into BOTH targets
    ├── AppGroup.swift                   # App Group identifier constant
    ├── KeychainHelper.swift             # App Group-scoped Keychain read/write
    ├── AnthropicClient.swift            # Messages API with prompt caching
    └── SystemPrompt.swift               # The IP — rewrite rules + 3 few-shot examples
```

### Prompt caching

`AnthropicClient.swift` sends the system prompt with `cache_control: ephemeral`. After the first call, subsequent calls hit the cache and the input cost drops ~90%. You can verify this is working by watching the status line under the polished output — `cache read` should be > 0 from your second call onward (within ~5 min of the previous call).

### Costs

- Default model: `claude-haiku-4-5` (~$1/M input, $5/M output)
- Typical polish call: ~1200 input tokens (system prompt + your text) + ~300 output tokens
- First call: ~$0.0027
- Cached calls: ~$0.0006
- 50 uses/day ≈ $1/month

### Editing the system prompt

The system prompt is the IP. As you use the app, keep a list of dictations that produced disappointing output. Each one becomes either a new rule or a new few-shot example in `Shared/SystemPrompt.swift`. Re-build, test. Don't tune the prompt in the abstract — tune it against real misses.

## Known limitations

- Keyboard extensions have a tight memory budget (~60 MB). The current implementation keeps it well under, but if you add features watch for OOM kills.
- The keyboard's "Polish" action grabs the entire visible text context (before+after cursor). For long fields this may grab more than you intended. Workaround: select the specific text you want polished before tapping Polish.
- Speech recognition in the host app uses Apple's on-device recognizer. Quality is good but accents and technical jargon get garbled — that's exactly the problem the polish step is designed to fix.

## Troubleshooting

- **"Provisioning profile doesn't include..."** — make sure both targets have the **App Groups** capability with the same group ID, and the group ID is registered in your developer account (Xcode usually does this automatically with Automatic signing).
- **Keyboard doesn't appear in the keyboard list** — re-run the app once, kill it, then check Settings.
- **"No API key set"** error from the keyboard** — open the host app, go to Settings, save the API key, then try the keyboard again. The Keychain entry is shared via App Group; if the App Group IDs don't match between the two targets, the keyboard won't see the key.
- **Keyboard taps do nothing** — check that **Allow Full Access** is enabled. Without it, the keyboard cannot make network calls.
